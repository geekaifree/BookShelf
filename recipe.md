# Tandoor Recipes：食谱管理系统

## 简介

Tandoor Recipes 是一款开源食谱管理系统，帮助用户组织、存储和分享烹饪食谱。它提供基于 Web 的界面，用于管理食谱、规划膳食、生成购物清单和追踪营养。系统支持多用户、从各种来源导入食谱以及丰富的分类选项。

**核心功能：**

- 食谱存储和组织
- 膳食规划
- 购物清单生成
- 营养追踪
- 从 URL 导入食谱
- 多用户支持
- 分类和标签
- 食谱分享
- 备份和恢复
- 移动端响应式界面

## 安装

### 系统要求

| 组件 | 最低 | 推荐 |
|-----------|---------|-------------|

| Docker Compose | 1.29+ | 最新 |

### Docker 安装（推荐）

创建 `docker-compose.yml` 文件：

```yaml
version: "3"
services:
  db_recipes:
    image: postgres:14-alpine
    restart: always
    volumes:
      - postgres_data:/var/lib/postgresql/data
    environment:
      - POSTGRES_PASSWORD=change_me
      - POSTGRES_DB=tandoor
      - POSTGRES_USER=tandoor

  web_recipes:
    image: vabene1111/recipes
    restart: always
    volumes:
      - static_files:/opt/recipes/staticfiles
      - nginx_config:/opt/recipes/nginx
      - media_files:/opt/recipes/media
    environment:
      - SECRET_KEY=change_me_to_random_string
      - DB_ENGINE=django.db.backends.postgresql
      - POSTGRES_HOST=db_recipes
      - POSTGRES_PORT=5432
      - POSTGRES_USER=tandoor
      - POSTGRES_PASSWORD=change_me
      - POSTGRES_DB=tandoor
      - ALLOWED_HOSTS=*

  nginx_recipes:
    image: nginx:mainline-alpine
    restart: always
    ports:
      - "8080:8080"
    volumes:
      - nginx_config:/etc/nginx/conf.d:ro
      - static_files:/static:ro
      - media_files:/media:ro

volumes:
  postgres_data:
  static_files:
  nginx_config:
  media_files:
```

启动服务：

```bash
docker-compose up -d
```

访问 `http://localhost:8080` 即可使用。

### 手动安装

依赖要求：

| 软件 | 版本 |
|----------|---------|
| Python | 3.8+ |
| PostgreSQL | 12+ |
| Node.js | 16+ |
| Redis | 6+ |

步骤：

```bash
git clone https://github.com/TandoorRecipes/recipes.git
cd recipes
pip install -r requirements.txt
python manage.py migrate
python manage.py collectstatic
python manage.py runserver
```

### 环境变量

| 变量 | 描述 | 默认值 |
|----------|-------------|---------|
| `SECRET_KEY` | Django 密钥 | 必填 |
| `DB_ENGINE` | 数据库后端 | postgresql |
| `POSTGRES_HOST` | 数据库主机 | db_recipes |
| `POSTGRES_PORT` | 数据库端口 | 5432 |
| `POSTGRES_USER` | 数据库用户 | tandoor |

| `POSTGRES_DB` | 数据库名称 | tandoor |
| `ALLOWED_HOSTS` | 允许的主机名 | * |
| `DEBUG` | 调试模式 | 0 |
| `GUNICORN_MEDIA` | 提供媒体文件 | 0 |

## 食谱管理

### 创建食谱

1. 点击"新食谱"
2. 填写食谱详情：
   - 名称
   - 描述
   - 做法
   - 食材
   - 准备时间
   - 烹饪时间
   - 份数
3. 添加图片
4. 设置分类和标签
5. 保存食谱

### 食谱字段

| 字段 | 描述 | 必填 |
|-------|-------------|----------|
| 名称 | 食谱标题 | 是 |
| 描述 | 简短描述 | 否 |
| 做法 | 分步说明 | 是 |
| 食材 | 食材列表 | 是 |
| 准备时间 | 准备时间 | 否 |
| 烹饪时间 | 烹饪时间 | 否 |
| 份数 | 份数 | 否 |
| 工作时间 | 主动烹饪时间 | 否 |
| 等待时间 | 被动时间（腌制等） | 否 |
| 来源 | 食谱来源 URL | 否 |
| 图片 | 食谱照片 | 否 |

### 食材

格式化食材，包含数量、单位和食物：

| 格式 | 示例 |
|--------|---------|
| 数量 + 单位 + 食物 | 2 杯面粉 |
| 数量 + 食物 | 3 个鸡蛋 |
| 备注 | 1 杯糖（最好是红糖） |

**支持的单位：**

| 类别 | 单位 |
|----------|-------|
| 容量 | 杯、汤匙、茶匙、ml、l、fl oz |
| 重量 | g、kg、oz、lb |
| 计数 | 个、瓣、片、束 |
| 其他 | 少许、适量 |

### 做法

编写清晰的分步说明：

1. 每步应是一个单独操作
2. 包含温度和时间
3. 注明重要细节
4. 使用一致的格式

## API 使用

### REST API

Tandoor 提供 REST API 用于编程访问：

**认证：**

```bash
curl -X POST "http://localhost:8080/api-token-auth/" \
  -H "Content-Type: application/json" \
  -d '{"username": "admin", "password": "password"}'
```

**列出食谱：**

```bash
curl "http://localhost:8080/api/recipe/" \
  -H "Authorization: Bearer <token>"
```

### API 端点

| 端点 | 方法 | 描述 |
|----------|--------|-------------|
| `/api/recipe/` | GET | 列出食谱 |
| `/api/recipe/{id}/` | GET | 获取食谱 |
| `/api/recipe/` | POST | 创建食谱 |
| `/api/recipe/{id}/` | PUT | 更新食谱 |
| `/api/recipe/{id}/` | DELETE | 删除食谱 |
| `/api/meal-plan/` | GET | 列出膳食计划 |
| `/api/shopping-list/` | GET | 列出购物清单 |

## 故障排除

### 常见问题

| 问题 | 解决方案 |
|-------|----------|
| 无法访问 | 检查端口、防火墙 |
| 导入失败 | 验证 URL 可访问性 |
| 图片缺失 | 检查媒体卷 |
| 性能缓慢 | 增加资源 |
| 数据库错误 | 检查 PostgreSQL 日志 |

### 性能调优

| 设置 | 建议 |
|---------|---------------|
| 数据库 | 定期清理 |
| 缓存 | 启用 Redis 缓存 |
| 媒体 | 使用 S3 存储 |
| 工作线程 | 增加 Gunicorn 工作线程 |
