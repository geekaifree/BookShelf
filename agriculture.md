## 自动化和机器人

- [Ant Robotics](https://antrobotics.de/) - 由 Ecoterra bot 开发
- [Acorn Rover](https://github.com/Twisted-Fields) - 精准农业漫游车，Odrive，Python。
- [地球漫游者](https://github.com/earthrover) - Ag AGV ROS1精准农业漫游车 
- [EcoTerra 机器人](https://ecoterrabot.com/) - Delta & Rover
- [FarmBot](https://github.com/farmbot) - 开源精准园艺项目。
- [FarmBot-ROS](https://github.com/farmbot-ros/) - 开源农业机器人。
- [Fields2Cover](https://github.com/Fields2Cover/Fields2Cover) - 强大而高效的自动农用车辆覆盖路径。
- [Field Friend](https://github.com/zauberzeug/field_friend) - 自主除草开发平台。 
- [Romi 项目](https://media.romi-project.eu/documents/index.html) - 欧洲资助的研究项目 
- [ROS Agriculture](http://rosagriculture.org/) - 开源社区，专注于使用机器人操作系统为农民提供机器人工具。 
- [除草器](https://hackaday.io/project/53896-weedinator-2019) - 巡线除草机器人

## 计算器

- [farm-calculators](https://github.com/brycejohnston/farm-calculators) - 用于计算各种农业和农作物相关信息的 WordPress 插件。

## 气候、环境和天气

- [agroclimatology](https://github.com/brycejohnston/agroclimatology) - 用于与 NASA (POWER) 农业气候学网络资源交互的 Ruby 客户端。
- [evapotranspiration](https://github.com/brycejohnston/evapotranspiration) - 用于计算参考作物蒸散量 (ETo) 的 Ruby 库。
- [frostline](https://github.com/waldoj/frostline) - 美国农业部植物抗寒区的数据集、API 和 python 解析器。
- [GSODR](https://github.com/ropensci/GSODR) - R 中的全球每日天气数据汇总。
- [iem](https://github.com/akrherz/iem) - 使爱荷华州环境 Mesonet 运行的代码。
- [PyETo](https://github.com/woodcrafty/PyETo) - 用于计算参考/潜在蒸散量 (ETo) 的 Python 包。
- [pyTSEB](https://github.com/hectornieto/pyTSEB) - 一个 python 两源能量平衡模型，用于利用遥感数据估计蒸散量。
- [soilDB](https://github.com/ncss-tech/soilDB) - R 库，用于简化对 NCSS 土壤数据库的访问。

## 作物建模、表型分析和病理学

- [开放植物病理学](https://www.openplantpathology.org/) - 一个重视开放数据和计算工具以推进流行病学和病原体种群生物学和生态学的社区。

## 数据集

- [农业数据集集合](https://lacunafund.org/datasets/agriculture/) - 精心策划的机器学习数据集集合，专门用于解决发展中地区的粮食安全挑战。
- [CEAOD](https://ceaod.github.io/) - 受控环境农业开放数据存储库为温室和室内农业研究提供标准化数据集和数据分析工具。
- [CropHarvest](https://github.com/nasaharvest/cropharvest) - 用于全球作物类型绘图的开源遥感数据集，包含来自卫星图像的 95,186 个标记数据点，支持粮食安全的机器学习研究。
- [CWFID](https://github.com/cwfid/dataset) - 数据集包括田野图像、植被分割掩模和作物/杂草植物类型注释。
- [FAO/NASA 收获作物类型数据集](https://data.harvestportal.org/dataset/fao-bha-nasa-harvest-public-crop-type-datasets) - 地面真实作物类型观测数据集，包含地理定位的现场数据，用于验证和训练农田分类模型。
- [Growstuff](https://www.growstuff.org/crops) - 记录保存和作物数据库，不错的 API
- [PlantVillage](https://www.kaggle.com/datasets/emmarex/plantdisease) - 包含健康和患病植物叶子图像的数据集，涵盖 14 种不同作物的 38 种疾病类别。
- [SoybeanYield](https://github.com/saeedkhaki92/CNN-RNN-Yield-Prediction?tab=readme-ov-file) - 用于大豆产量预测的多模式农业数据集，集成了产量绩效、管理实践、天气数据和土壤参数。 
- [TERRA REF](https://terraref.org) - 来自高粱育种试验的 1PB 公共领域高分辨率传感器数据（[数据发布](https://doi.org/10.5061/dryad.4b8gtht99)，大文件可在 [globus.org 上获取] ncsa#terra-public](https://app.globus.org/file-manager?origin_id=e8feaff4-96cd-11ea-bf90-0e6cccbb0103&origin_path=%2F))

## 数据标准化、互操作性和 API

- [AgGateway 的 ADAPT 工具包](https://adaptframework.org) - 开源项目提供工具来简化种植者、机器和合作伙伴之间的通信。
- [Agstack](https://github.com/agstack) - 农业生态系统的开源数字基础设施。
- [API-Code-Samples](https://github.com/aWhereAPI/API-Code-Samples) - 包含 aWhere 的 API 平台代码示例。
- [Botanica](https://github.com/Michael-A-Kuykendall/botanica) - 用于专业植物数据管理的生产就绪 Rust 箱，具有分类层次结构、栽培跟踪和 Darwin Core 合规性。
- [ClearAg-API-Examples](https://github.com/IterisClearAg/ClearAg-API-Examples) - 使用 ClearAg API 的代码示例集合。
- [Fiware AgriFood](https://www.fiware.org/community/smart-agrifood/) - 开源数据标准和物联网框架，有助于连接农场系统并实现实用的智能农业和农业食品解决方案 - 是更大的 Fiware.org 项目的一部分。
- [JDLinkMachineDataAPI-OAuth2-CSharp-Example](https://github.com/JohnDeere/JDLinkMachineDataAPI-OAuth2-CSharp-Example) - CSharp JDLink 机器数据 API 客户端示例。
- [开放农业数据联盟](https://github.com/oada) - OADA 是一个开放项目，旨在为农业数据带来互操作性、安全性和隐私性。
- [MyJohnDeereAPI-OAuth2-Java-Example](https://github.com/JohnDeere/MyJohnDeereAPI-OAuth2-Java-Example) - Java MyJohnDeere API 客户端示例。
- [MyJohnDeereAPI-OAuth2-NodeJS-Example](https://github.com/JohnDeere/MyJohnDeereAPI-OAuth2-NodeJS-Example) - Node.js MyJohnDeere API 客户端示例。
- [MyJohnDeereAPI-OAuth2-Python-Example](https://github.com/JohnDeere/MyJohnDeereAPI-OAuth2-Python-Example) - Python MyJohnDeere API 客户端示例。
- [OpenTeam](https://openteam.community/) - 农业管理开放技术生态系统
- [SampleData](https://github.com/JohnDeere/SampleData) - 示例数据卡、shapefile 和其他可用于测试的文件。
- [SencropAPI-JavaScript-Client](https://github.com/sencrop/sencrop-js-api-client) - Sencrop API Javascript 客户端实现。阅读和管理 Sencrop 气象站。

## 农场管理系统和记录保存

- [Ekylibre](https://github.com/ekylibre/ekylibre) - 使用 Ruby on Rails 和 PostgreSQL/PostGIS 构建的面向农民和小型企业的农场管理信息系统。
- [farmOS](https://github.com/farmOS/farmOS) - 使用 Drupal 和 PHP 构建的基于 Web 的农场记录保存应用程序。
- [GrowGood](https://gitlab.com/growgood)) - GrowGood 是一个开源农场管理平台，使用 ValueFlows 和 JSON-LD 等开放标准来帮助跟踪和共享跨农业运营和市场的可互操作数据。
- [LiteFarm](https://github.com/LiteFarmOrg/LiteFarm) - LiteFarm 是世界上第一个社区主导的、非营利性的农场管理系统
- [Soil Mate](https://github.com/Open-Source-Agriculture/soil_mate) - 旨在协助收集土壤数据的助手应用程序。
- [Tania](https://github.com/Tanibox/tania-core) - 使用 Go 和 Vue.js 构建的免费开源农业管理系统，适合所有人。
