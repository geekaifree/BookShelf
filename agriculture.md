## 数据集

- [农业数据集集合](https://lacunafund.org/datasets/agriculture/) - 精心策划的机器学习数据集集合，专门用于解决发展中地区的粮食安全挑战。
- [CEAOD](https://ceaod.github.io/) - 受控环境农业开放数据存储库为温室和室内农业研究提供标准化数据集和数据分析工具。
- [CropHarvest](https://github.com/nasaharvest/cropharvest) - 用于全球作物类型绘图的开源遥感数据集，包含来自卫星图像的 95,186 个标记数据点，支持粮食安全的机器学习研究。
- [CWFID](https://github.com/cwfid/dataset) - 数据集包括田野图像、植被分割掩模和作物/杂草植物类型注释。
- [FAO/NASA 收获作物类型数据集](https://data.harvestportal.org/dataset/fao-bha-nasa-harvest-public-crop-type-datasets) - 地面真实作物类型观测数据集，包含地理定位的现场数据，用于验证和训练农田分类模型。
- [Growstuff](https://www.growstuff.org/crops) - 记录保存和作物数据库，不错的 API
- [PlantVillage](https://www.kaggle.com/datasets/emmarex/plantdisease) - 包含健康和患病植物叶子图像的数据集，涵盖 14 种不同作物的 38 种疾病类别。
- [SoybeanYield](https://github.com/saeedkhaki92/CNN-RNN-Yield-Prediction?tab=readme-ov-file) - 用于大豆产量预测的多模式农业数据集，集成了产量绩效、管理实践、天气数据和土壤参数。 
- [TERRA REF](https://terraref.org) - 来自高粱育种试验的 1PB 公共领域高分辨率传感器数据（[数据发布](https://doi.org/10.5061/dryad.4b8gtht99)，大文件可在 [globus.org 上获取] ncsa#terra-public](https://app.globus.org/file-manager?origin_id=e8feaff4-96cd-11ea-bf90-0e6cccbb0103&origin_path=%2F))

## 简介

OpenFarm 是一个开源平台，用于分享和发现种植知识。它提供了结构化的种植指南、作物信息和最佳实践数据库，适合各种经验水平的园丁和农民。

### 平台概览

| 功能 | 描述 |
|------|------|
| 种植指南 | 作物种植的分步指导 |
| 作物数据库 | 全面的植物信息 |
| 用户社区 | 共享经验和建议 |
| API 访问 | 编程方式访问数据 |
| 开源 | 透明、社区驱动 |

### 核心理念

OpenFarm 坚信：

- **开放知识**：免费获取种植信息
- **社区驱动**：全球农民的集体智慧
- **实用导向**：可操作、经过验证的建议
- **生物多样性**：支持多样作物品种
- **可持续性**：环境友好的实践

### 架构

```
OpenFarm 数据结构
├── 作物
│   ├── 名称和分类
│   ├── 种植要求
│   ├── 常见品种
│   └── 图片

│   ├── 前置条件
│   ├── 种植指导
│   ├── 养护要求
│   └── 收获时机
├── 花园
│   ├── 用户花园设置
│   ├── 已种作物
│   └── 活动日志
└── 社区
    ├── 用户资料
    ├── 评论
    └── 评分
```

## 种植指南

### 指南结构

每份种植指南包含标准化章节：

| 章节 | 内容 |
|------|------|
| 概述 | 简要描述和目标 |
| 前置条件 | 开始前需要准备什么 |
| 时间线 | 何时种植和收获 |
| 步骤 | 详细指导 |
| 问题排查 | 常见问题和解决方案 |
| 技巧 | 专家建议 |

### 示例：番茄种植指南

**概述：**
番茄是暖季作物，整个夏季都能结果。需要充足阳光、持续浇水和支撑结构。

**前置条件：**

| 要求 | 详情 |
|------|------|
| 阳光 | 6-8 小时直射阳光 |
| 土壤 | 排水良好，pH 6.0-6.8 |
| 间距 | 植株间 24-36 英寸 |
| 支撑 | 笼子、支架或棚架 |
| 气候 | 暖季，无霜 |

**时间线：**

| 阶段 | 时间 |
|------|------|
| 室内育苗 | 末次霜冻前 6-8 周 |
| 移栽 | 末次霜冻后，土温 > 60 deg F |
| 首次结果 | 移栽后 60-80 天 |
| 收获 | 移栽后 80-120 天 |

**步骤：**

1. **育苗**
   - 在育苗盘中填入育苗基质
   - 种子播深 1/4 英寸
   - 保持湿润和温暖（70-80 deg F）
   - 提供 12-16 小时光照

2. **移栽**
   - 炼苗 7-10 天
   - 比在盆中种得更深（埋入的茎会长出根）
   - 种后充分浇水
   - 基部覆盖覆盖物

3. **日常养护**
   - 每周深浇 1-2 英寸水量
   - 无限生长型品种需打侧枝
   - 每 2-3 周施肥
   - 监测病虫害

| USDA 区 | 温度范围 | 适宜作物 |
|---------|----------|----------|

## 土壤准备

### 土壤检测

种植前检测土壤：

| 检测项 | 理想范围 | 偏低修正 | 偏高修正 |
|--------|----------|----------|----------|
| pH | 6.0-7.0 | 石灰 | 硫磺 |
| 氮 | 20-40 ppm | 堆肥、血粉 | 减少施肥 |
| 磷 | 25-50 ppm | 骨粉 | 减少施肥 |
| 钾 | 150-250 ppm | 草木灰、绿砂 | 水洗 |
| 有机质 | 3-5% | 堆肥 | 不适用 |

### 土壤改良指南

| 改良剂 | 用途 | 施用量 |
|--------|------|--------|
| 堆肥 | 改善结构、增加养分 | 2-4 英寸/年 |
| 沤熟粪肥 | 增加养分、有机质 | 1-2 英寸/年 |
| 石灰 | 提高 pH | 5-10 磅/100 平方英尺 |
| 硫磺 | 降低 pH | 1-2 磅/100 平方英尺 |
| 珍珠岩 | 改善排水 | 混合 10-20% |
| 蛭石 | 改善保水 | 混合 10-20% |
| 泥炭苔 | 酸化、保水 | 混合 20-30% |
| 骨粉 | 增加磷 | 5-10 磅/100 平方英尺 |

### 土壤结构改善

| 问题 | 症状 | 解决方案 |
|------|------|----------|
| 板结 | 排水差、难以挖掘 | 添加有机质、宽叉松土 |
| 黏土 | 湿时粘、干时裂 | 添加石膏、堆肥 |

＃＃ 入门

养蜂需要准备、投资和持续关注。在订购蜜蜂之前，请了解承诺：蜂巢需要定期检查（活跃季节每 1-2 周一次），蜜蜂需要全年管理。

### 法律要求

|要求 |详情 |
|-------------|---------|
|地方法规|检查市/县分区法。一些城市禁止或限制后院蜂巢。 |
|注册 |大多数州和省要求养蜂人向农业部门注册蜂箱。 |
|后退距离|许多地区要求蜂巢与地产红线、道路或邻近建筑物保持最小距离（通常为 10-25 英尺）。 |
|水源|您必须提供水源，以免蜜蜂访问邻居的水池、宠物碗或喷泉。 |
|邻居通知 |法律并不总是要求这样做，但强烈建议这样做。在问题变成投诉之前解决它们。 |

## 所需设备

### 必备装备

|项目 |目的|大约成本（美元）|
|------|---------|----------------------|
|蜂巢体（盒子、框架、基础）|蜜蜂的家| 150-300 美元 |
|蜜蜂套装或带面纱的夹克|防止蜇伤 | 40-150 美元 |
|手套|手部防护 | 15-40 美元 |
|蜂巢工具 |撬开框架，刮掉蜡和蜂胶| 10-15 美元 |
|吸烟者 |检查期间安抚蜜蜂 | 25-50 美元 |
|蜜蜂刷|轻轻地将蜜蜂移离框架 | 5-10 美元 |
|蜜蜂（群体或核心群体）|殖民地本身| 120-250 美元 |

### 可选但有帮助

- 框架握把更容易操作。
- 入口减速器可帮助小型蜂群保卫蜂巢。
- 用于补充喂养的喂食器（顶部喂食器、入口喂食器或框架喂食器）。
- 蜂王排除器，使蜂王远离蜂蜜超级。

## 蜂巢类型

|类型 |设计|优点 |缺点 |
|------|--------|------|------|
| **兰斯特罗斯** |带可拆卸框架的可堆叠矩形盒子|行业标准；广泛可用；容易找到建议和设备；可扩展；良好的蜂蜜生产|重箱（满 50-60 磅）；蜜蜂在地基上筑巢；自然细胞尺寸较小|
| **顶栏** |水平蜂巢，顶部有横条；没有框架 |收获较轻（一次一根）；无需搬运重物；蜜蜂建造天然蜂巢；降低建造成本|蜂蜜产量减少；梳子易碎；在不破坏梳子的情况下更难以检查；有限扩张|
| **流蜂巢** |改良的 Langstroth 框架可打开以排出蜂蜜 |无需打开蜂巢即可收获蜂蜜；最大限度减少蜜蜂干扰；适合初学者 |昂贵（$500-700+）；仍需定期检查；框架仅适用于超级蜂蜜，不适用于育雏箱 |

对于大多数初学者来说，建议使用 **10 帧 Langstroth 蜂箱**。它是记录最多的系统，当地养蜂人、供应商店和在线社区绝大多数都使用它。

## 蜜蜂生物学和行为

### 群体结构

|角色 |每个集落的数量 |功能|
|------|-------------------|----------|
|女王 | 1 |产蛋（旺季最多 2,000 个/天）；产生调节蜂群行为的信息素
|工人| 20,000-80,000 |女性；执行所有任务：觅食、护理、筑巢、看守、温度调节|
|无人机 | 0-1,000 |男性；它们的唯一目的是与其他殖民地的蚁后交配。它们交配后就会死亡。 |

|舞台|持续时间 |
|-------|----------|
|蛋| 3 天 |
|幼虫（由护士蜂喂养）| 6 天 |
|蛹（密封在细胞中）| 12 天 |
|出现|总计：从卵到成虫 21 天 |

### 按年龄划分的工人任务

|年龄（天）|任务|
|------------|------|
| 1-3 | 1-3清洁牢房以供女王躺卧 |
| 3-6 | 3-6喂幼虫蜂王浆，然后喂花粉和蜂蜜|
| 6-12 |生产蜡，打造梳子|
| 12-18 |接收和处理花蜜，守卫入口|

### 读取蜜蜂行为

|行为 |意义|
|----------|---------|
|框架上平静、稳定的运动 |健康的皇后右翼殖民地 |
|蜜蜂在入口处扇风（腹部向上）|冷却蜂巢或分散纳索诺夫信息素以引导蜜蜂回家
|蜜蜂成群地挂在入口外|天气炎热；可能过度拥挤（可能蜂拥而至）|

|高音嗡嗡声、低头姿势|防守。谨慎行事或使用更多烟雾。 |

## 季节性管理

＃＃＃ 春天

- 冬天过后检查是否有活着的蚁后（可见卵）。
- 当第一个育雏箱已充满 70-80% 的蜜蜂和育雏时，添加第二个育雏箱。
- 观察蜂群的准备情况（蜂王室位于框架底部）。
- 如果还没有天然花蜜，则开始喂食 1:1 糖浆。
- 用新粉底更换旧的深色梳子。

＃＃＃ 夏天

- 当花蜜开始流动时添加蜂蜜。
- 确保在炎热天气下有足够的通风（稍微支撑外罩）。
- 每 7-10 天监测一次蜂群情况。
- 保持定期检查。

＃＃＃ 落下

- 收获蜂蜜（留下足够的蜂蜜让蜂群过冬——通常为 60-90 磅，具体取决于气候）。
- 治疗瓦螨（参见疾病和害虫部分）。
- 减少蜂巢入口，帮助蜂群抵御强盗和黄衫军。
- 如果储备不足，开始喂食 2:1 糖浆。

＃＃＃ 冬天

- 除非绝对必要，否则不要打开蜂巢。
- 确保通风以防止水分积聚（水分比寒冷杀死更多的菌落）。
- 无论外部温度如何，蜂群内部温度都保持在约 93 华氏度 (34 摄氏度)。
- 通过从后面举起蜂巢定期检查食物储备。
- 在入口处放置防鼠装置。

## 常见病虫害

### 瓦螨破坏螨

全球受管理蜜蜂面临的最严重威胁。

|方面|详情 |
|--------|---------|
|它是什么 |一种以蜜蜂脂肪体为食并传播病毒的外寄生螨虫 |
|检测|用酒精清洗或糖卷约 300 只蜜蜂。阈值：每 100 只蜜蜂 >2-3 只螨虫需要治疗。 |
|治疗方案|草酸（蒸发或滴加）、甲酸（除螨带）、Apiguard（百里酚）、Apivar（双甲脒）、雄蜂除巢 |
|监控时间表|从春季到秋季每月进行一次测试 |

### 其他常见问题

|问题 |症状 |管理 |
|---------|----------|------------|
|美国臭虫 (AFB) |凹陷、油腻的盖子；粘稠的棕色幼虫；恶臭气味|向当局报告。烧蜂巢。传染性极强。 |
|欧洲腐殖质 (EFB) |扭曲、变色的幼虫；酸臭味|雷王。可能会以强大的菌落恢复。 |
|微孢子虫 |痢疾（蜂巢正面有棕色条纹）；弹簧积聚不良|雷奎恩；确保通风良好；夫马洁林在哪里合法|
|小蜂巢甲虫 |蜂巢中的成虫；粘糊糊的、破烂的梳子|甲虫陷阱；强大的群体可以更好地管理它们；减少蜂巢空间|
|蜡蛾|梳子中的带子和通道，特别是存储的梳子|将未使用的梳子与对二氯苯晶体一起存放；冷冻梳|

## 蜂蜜收获

### 何时收获

当蜜蜂用蜡盖住蜂房时，蜂蜜就准备好了。未封盖的蜂蜜水分含量过高，会发酵。

＃＃＃ 过程

1. 将蜜蜂从框架（蜂刷、通风板或蜜蜂逃生装置）中移走。
2. 用开盖刀或叉子打开覆盖蜜细胞的蜡。
3. 将框架放入提取器（离心旋转器）中以旋转出蜂蜜。
4. 用网状过滤器过滤蜂蜜，除去蜡颗粒。
5. 让蜂蜜在桶中静置 24-48 小时。气泡和泡沫上升到顶部。
6. 装入干净的玻璃罐中。

### 蜂蜜品种

|品种 |颜色 |风味 |来源 |
|---------|-------|--------|--------|
|三叶草|浅琥珀色|温和、经典的“蜜”味|三叶草花|
|野花|中琥珀色|因地区和季节而异 |混合野花|
|荞麦|黑暗|浓郁、麦芽味、糖蜜般 |荞麦花|
|相思|非常轻，几近透明|细腻、温和|黑槐花开|
|麦卢卡 |深琥珀色|土质、草本 |麦卢卡灌木（新西兰）|
|橙花|浅琥珀色|柑橘、花香|柑橘园|

蜂蜜的颜色和味道完全取决于蜜蜂可获得的花蜜来源。当地的时令蜂蜜是该地区独有的。

除了蜂蜜生产之外，蜜蜂还提供重要的授粉服务。人类吃的食物中大约有三分之一依赖于蜜蜂和其他昆虫的授粉。

|作物|对蜜蜂授粉的依赖|
|------|-------------------------------|

|蓝莓| 90%+ |
|苹果 | 80-90% |

|南瓜/南瓜 | 70-90% |

许多养蜂人通过在开花季节将蜂巢放置在农田中来赚取额外收入。

## 第一年的期望

您的第一年是学习，而不是最大化蜂蜜产量。

- 春季安装的核心菌落 (nuc) 可能会积累到足以在秋季生产一些剩余的蜂蜜，但这并不能保证。
- 专注于保持殖民地的活力和健康。
- 预计会犯错误。经验丰富的养蜂人仍然会失去蜂群。
- 加入当地的养蜂俱乐部。实践指导远比单纯的书本更有价值。
- 第一年预算 500-800 美元，包括蜜蜂、设备和防护装备。

### 天气 API 示例

```python
import requests

def get_weather(lat, lon, api_key):
    url = f"https://api.openweathermap.org/data/2.5/weather"
    params = {
        "lat": lat,
        "lon": lon,
        "appid": api_key,
        "units": "metric"
    }
    response = requests.get(url, params=params)
    data = response.json()

    return {
        "temperature": data["main"]["temp"],
        "humidity": data["main"]["humidity"],
        "description": data["weather"][0]["description"],
        "wind_speed": data["wind"]["speed"]
    }
```

### 处理 Sentinel-2 数据

```python
import rasterio
import numpy as np

def calculate_ndvi(nir_band, red_band):
    """从近红外和红波段计算 NDVI。"""
    nir = nir_band.astype(float)
    red = red_band.astype(float)
    ndvi = (nir - red) / (nir + red)
    return ndvi

# 读取 Sentinel-2 波段
with rasterio.open("B04.tif") as red_src:
    red = red_src.read(1)
with rasterio.open("B08.tif") as nir_src:
    nir = nir_src.read(1)

ndvi = calculate_ndvi(nir, red)
```

import paho.mqtt.client as mqtt
import json

def on_message(client, userdata, message):
    payload = json.loads(message.payload.decode())
    topic = message.topic

    if "soil_moisture" in topic:
        print(f"土壤湿度：{payload['value']}%")
    elif "temperature" in topic:
        print(f"温度：{payload['value']}°C")

### 病害检测示例

```python
import tensorflow as tf
from tensorflow.keras.preprocessing.image import ImageDataGenerator

# 加载预训练模型
model = tf.keras.models.load_model("crop_disease_model.h5")

# 从叶片图像预测病害
def predict_disease(image_path):
    img = tf.keras.preprocessing.image.load_img(
        image_path, target_size=(224, 224)
    )
    img_array = tf.keras.preprocessing.image.img_to_array(img)
    img_array = tf.expand_dims(img_array, 0) / 255.0

    predictions = model.predict(img_array)
    classes = ["Healthy", "Blight", "Rust", "Mildew"]
    return classes[predictions.argmax()]
```
