## 简介

Lichess 是一个免费、开源的国际象棋服务器，提供无广告、无付费功能的完整国际象棋体验。由社区构建，提供从休闲对局到专业锦标赛的一切功能。

### 核心优势

| 功能 | 描述 |
|------|------|
| 100% 免费 | 无广告，无付费层级，所有功能免费 |
| 开源 | 代码透明，社区贡献 |
| 无需注册 | 可匿名对局 |
| 强大的分析功能 | 所有对局均可免费使用 Stockfish 分析 |
| 多种变体 | 标准国际象棋及创意变体 |
| 全球社区 | 来自世界各地的玩家 |
| 移动应用 | iOS 和 Android 应用 |
| API 访问 | 完整的 REST 和 WebSocket API |

### 平台统计

| 指标 | 数值 |
|------|------|
| 注册用户 | 1000 万+ |
| 每日对局数 | 100 万+ |
| 活跃玩家 | 每日 5 万+ |
| 可用谜题 | 20 万+ |
| 支持语言 | 80+ |

### 快速开始

1. 访问 lichess.org
2. 匿名对局或创建账户
3. 开始对局、解答谜题或学习
4. 在个人资料中跟踪进度

**每日谜题：**

- 每天一个精选谜题
- 所有用户相同的谜题
- 评论区讨论

**Puzzle Storm：**

- 3 分钟内尽可能多地解题

**Puzzle Racer：**

- 与其他玩家竞速
- 首先正确解答者获胜

## 国际象棋变体

### 可用变体

| 变体 | 描述 | 棋盘/棋子 |
|------|------|-----------|
| 标准 | 常规国际象棋 | 8x8，标准棋子 |
| Chess960 | 随机起始局面 | 8x8，打乱的底线 |
| 山丘之王 | 到达中心获胜 | 8x8，标准 |
| 三次将军 | 将军 3 次获胜 | 8x8，标准 |
| 反棋 | 必须吃子，失去所有棋子获胜 | 8x8，标准 |
| 原子棋 | 吃子引发爆炸 | 8x8，标准 |
| 军团棋 | 兵对棋子 | 8x8，修改版 |
| 竞速王 | 将王跑到第 8 横线 | 8x8，标准 |
| Crazyhouse | 放置被吃棋子 | 8x8，标准 |

### Chess960（费舍尔随机棋）

| 方面 | 描述 |
|------|------|
| 起始局面 | 底线棋子随机排列 |
| 规则 | 与标准国际象棋相同 |
| 王车易位 | 修改的王车易位规则 |
| 目的 | 减少开局理论 |

**起始局面生成：**

1. 将象放在不同颜色的格子上
2. 将后放在剩余格子上
3. 将马放在剩余格子上
4. 将车和王放在剩余格子上（王在车之间）

### 三次将军棋

| 规则 | 描述 |
|------|------|
| 获胜条件 | 将军对手 3 次 |
| 标准规则 | 所有常规规则适用 |
| 策略 | 激进玩法受奖励 |
| 时间控制 | 通常更快 |

### 反棋（让子棋）

| 规则 | 描述 |
|------|------|
| 必须吃子 | 如果可能必须吃子 |
| 获胜条件 | 失去所有棋子（或困住对手） |
| 王 | 无特殊地位 |
| 升变 | 可以升变为王 |

### 原子棋

| 规则 | 描述 |
|------|------|
| 吃子爆炸 | 被吃棋子和周围棋子（除兵外）爆炸 |
| 获胜条件 | 炸掉对手的王 |
| 王的安全 | 关键考虑因素 |
| 策略 | 战术性、牺牲性玩法 |

### Crazyhouse

| 规则 | 描述 |
|------|------|
| 放置棋子 | 将被吃棋子放在空格上 |
| 兵的限制 | 不能在第 1/8 横线放置 |
| 将军 | 放置的棋子可以将军 |
| 策略 | 物质 vs. 棋子活性 |

## API 文档

### API 概览

Lichess 提供全面的 REST 和 WebSocket API，用于访问所有平台功能。

### 认证

**个人访问令牌：**

1. 前往 lichess.org/account/oauth/token
2. 选择权限范围
3. 生成令牌
4. 在 API 请求中使用

**OAuth2 流程：**

```python
import requests

# Authorization URL
auth_url = "https://lichess.org/oauth"
redirect_uri = "https://yourapp.com/callback"
client_id = "your_app_name"

# Exchange code for token
token_url = "https://lichess.org/api/token"
data = {
    'grant_type': 'authorization_code',
    'code': authorization_code,
    'redirect_uri': redirect_uri,
    'client_id': client_id
}
response = requests.post(token_url, data=data)
token = response.json()['access_token']
```

### Python 示例

**获取用户信息：**

```python
import requests

def get_user_info(username):
    url = f"https://lichess.org/api/user/{username}"
    headers = {"Accept": "application/json"}
```

```python
def get_game_pgn(game_id):
    url = f"https://lichess.org/game/export/{game_id}"
    headers = {"Accept": "application/x-chess-pgn"}
    response = requests.get(url, headers=headers)
```

### WebSocket API

**连接对局流：**

```python
import websocket
import json

def on_message(ws, message):
    data = json.loads(message)
    if 'chatLine' in data:
        print(f"Chat: {data['chatLine']['text']}")
    elif 'gameState' in data:
        print(f"Game state: {data['gameState']}")

def on_open(ws):
    # Authenticate
    ws.send(json.dumps({
        "t": "connect",
        "d": {
            "token": "YOUR_TOKEN",
            "gameId": "GAME_ID"
        }
    }))

ws = websocket.WebSocketApp(
    "wss://lichess.org/api/ws/game/GAME_ID",
    on_message=on_message,
    on_open=on_open
)
ws.run_forever()
```

### FEN 和 PGN

**FEN（Forsyth-Edwards 记谱法）：**

```
Starting position:
rnbqkbnr/pppppppp/8/8/8/8/PPPPPPPP/RNBQKBNR w KQkq - 0 1

Format:
{pieces} {turn} {castling} {en passant} {halfmove} {fullmove}
```

**PGN（便携式对局记谱法）：**

```
[Event "Casual Game"]
[Site "lichess.org"]
[Date "2024.01.15"]
[White "Player1"]
[Black "Player2"]
[Result "1-0"]

1. e4 e5 2. Nf3 Nc6 3. Bb5 a6 4. Ba4 Nf6 5. O-O Be7 1-0
```

## 引擎评估

### Stockfish 集成

Lichess 使用 Stockfish（最强的国际象棋引擎之一）进行分析。

| 设置 | 默认值 | 范围 | 描述 |
|------|--------|------|------|
| 深度 | 20 | 1-30 | 搜索深度 |
| Multi-PV | 1 | 1-5 | 显示的路线数 |
| 线程 | 1 | 1-16 | CPU 线程 |
| 哈希 | 16 | 1-1024 | 内存（MB） |

### 阅读引擎评估

**百分兵分数：**

| 分数 | 解读 |
|------|------|
| 0 | 局面均等 |
| +50 | 略优（白方） |
| +100 | 明显优势（白方） |
| +200 | 胜势（白方） |
| +500 | 决定性优势（白方） |
| -50 | 略优（黑方） |
| -100 | 明显优势（黑方） |
| -200 | 胜势（黑方） |
| -500 | 决定性优势（黑方） |

**将杀分数：**

| 分数 | 解读 |
|------|------|
| +M1 | 白方 1 步将杀 |
| +M5 | 白方 5 步将杀 |
| -M1 | 黑方 1 步将杀 |
| -M5 | 黑方 5 步将杀 |

### 评估图表

评估图表显示对局过程中局面的变化：

```
Evaluation Graph:

+2.0 |         ●
+1.0 |    ●         ●
 0.0 | ●              ●    ●
-1.0 |                    ●
-2.0 |                      ●
     +-------------------------> moves
```

- **上升线**：白方获得优势
- **下降线**：黑方获得优势
- **急剧变化**：战术失误或妙手
- **平坦线**：局面均等

### 云端评估

Lichess 为常见局面提供云端评估：

| 功能 | 描述 |
|------|------|
| 预计算 | 常见局面更深入评估 |
| 即时 | 无需等待本地计算 |
| 多路线 | 显示最佳走法 |
| 深度 | 40+ 深度 |

### 本地 vs 云端分析

| 方面 | 本地 | 云端 |
|------|------|------|
| 速度 | 取决于硬件 | 缓存内容即时响应 |
| 深度 | 用户控制 | 常见局面深度分析 |
| 隐私 | 完全私密 | 局面发送到服务器 |
| 可用性 | 始终可用 | 取决于缓存 |

### 分析工作流程

1. **快速复盘**：检查败着（深度 15）
2. **关键局面**：深度分析（深度 22+）
3. **多路线**：查看替代走法（Multi-PV 3）
4. **注释**：添加评论和符号
5. **研究**：创建研究以供长期学习

### 最佳实践

- 使用深度 20+ 进行正式分析
- 检查关键局面的多路线
- 不要完全依赖引擎评估
- 理解走法背后的计划
- 先在没有引擎的情况下练习局面
- 使用引擎验证人工分析
