## Transformer 架构

### 核心组件

| 组件 | 用途 | 关键机制 |
|-----------|---------|---------------|
| 嵌入 | 将 token 转换为向量 | 学习表示 |
| 位置编码 | 编码序列位置 | 正弦或学习 |
| 自注意力 | 捕获 token 关系 | Query、Key、Value |
| 前馈网络 | 转换表示 | 两个线性层 |
| 层归一化 | 稳定训练 | 归一化激活 |
| 残差连接 | 使深层网络成为可能 | 跳跃连接 |

### 自注意力机制

自注意力机制计算：

1. **Query（Q）：** token 在寻找什么
2. **Key（K）：** token 包含什么
3. **Value（V）：** 要传播的信息

**注意力公式：**
```
Attention(Q, K, V) = softmax(QK^T / sqrt(d_k)) * V
```

### 多头注意力

| 方面 | 描述 |
|--------|-------------|
| 多头 | 并行注意力计算 |
| 不同子空间 | 每个头学习不同模式 |
| 拼接 | 合并所有头 |
| 投影 | 线性变换到输出 |

### 模型架构

| 架构 | 描述 | 使用场景 |
|--------------|-------------|----------|
| 仅编码器 | BERT 风格 | 分类、NER |
| 仅解码器 | GPT 风格 | 文本生成 |
| 编码器-解码器 | T5 风格 | 翻译、摘要 |

## 分词

### 分词方法

| 方法 | 描述 | 示例 |
|--------|-------------|---------|
| 词级 | 按空格分割 | "Hello world" -> ["Hello", "world"] |
| 字符级 | 单个字符 | "Hello" -> ["H", "e", "l", "l", "o"] |
| BPE | 字节对编码 | 合并频繁出现的对 |
| WordPiece | 类似 BERT | "playing" -> ["play", "##ing"] |
| SentencePiece | 语言无关 | 处理任何语言 |
| Unigram | 概率性 | 选择最可能的分割 |

### 字节对编码（BPE）

1. 从单个字符开始
2. 找到最频繁的对
3. 将该对合并为新 token
4. 重复直到达到词汇表大小

**优势：**
- 处理未知词
- 平衡词汇表大小和序列长度
- 语言无关

### 分词器对比

| 分词器 | 模型 | 词汇量 | 特殊功能 |
|-----------|-------|------------|------------------|
| GPT-2 BPE | GPT-2 | 50,257 | 字节级 BPE |
| SentencePiece | Llama、Mistral | 32,000 | 语言无关 |
| WordPiece | BERT | 30,522 | 基于哈希 |
| tiktoken | GPT-3.5/4 | 100,000+ | 优化的 BPE |

## 预训练

### 预训练目标

| 目标 | 描述 | 模型 |
|-----------|-------------|--------|
| 因果语言模型 | 预测下一个 token | GPT、Llama |
| 掩码语言模型 | 预测被掩码的 token | BERT、RoBERTa |
| Seq2Seq | 从源生成目标 | T5、BART |

### 训练数据

| 来源 | 大小 | 质量 |
|--------|------|---------|
| Common Crawl | PB | 低（需要过滤） |
| Wikipedia | GB | 高 |
| 书籍 | GB | 高 |
| 代码 | GB | 中高 |
| 学术论文 | GB | 高 |

### 训练超参数

| 参数 | 典型范围 | 影响 |
|-----------|---------------|--------|
| 学习率 | 1e-4 到 3e-4 | 收敛速度 |
| 批量大小 | 1M-4M token | 训练稳定性 |
| 预热步数 | 1000-5000 | 防止早期不稳定 |
| 权重衰减 | 0.01-0.1 | 正则化 |
| 梯度裁剪 | 1.0 | 防止梯度爆炸 |

### 缩放定律

| 因素 | 关系 |
|--------|-------------|
| 模型大小 | 更多参数 = 更好性能 |
| 数据大小 | 更多数据 = 更好性能 |
| 计算量 | 更多 FLOPs = 更好性能 |
| Chinchilla | 最优比例：每个参数 20 个 token |

## 微调

### 全量微调

在特定任务数据上更新所有模型参数：

| 方面 | 描述 |
|--------|-------------|
| 数据 | 特定任务的标注数据集 |
| 计算量 | 高（所有参数更新） |
| 质量 | 最佳性能 |
| 灾难性遗忘 | 可能发生 |

### 参数高效微调（PEFT）

| 方法 | 描述 | 可训练参数 |
|--------|-------------|---------------------|
| LoRA | 低秩适应 | < 1% |
| QLoRA | 量化 LoRA | < 1% |
| 前缀调优 | 添加前缀 token | 约 0.1% |
| Adapter | 插入适配器层 | 约 1-5% |
| IA3 | 注入式适配器 | 约 0.01% |

### LoRA（低秩适应）

**工作原理：**
1. 冻结原始模型权重
2. 添加低秩分解矩阵
3. 仅训练低秩矩阵
4. 推理时合并

**超参数：**

| 参数 | 典型值 | 影响 |
|-----------|---------------|--------|
| Rank（r） | 8-64 | 容量 |
| Alpha | 16-128 | 缩放因子 |
| Target Modules | q_proj、v_proj | 适应哪些层 |
| Dropout | 0.05-0.1 | 正则化 |

### QLoRA

量化 LoRA 结合 4 位量化和 LoRA：

| 功能 | 描述 |
|---------|-------------|
| 4-bit NormalFloat | 量化基础模型 |
| 双重量化 | 量化量化常数 |
| 分页优化器 | 处理内存峰值 |

## RLHF（基于人类反馈的强化学习）

### RLHF 流程

| 步骤 | 描述 |
|------|-------------|
| 1. SFT | 基于演示的监督微调 |
| 2. 奖励模型 | 基于人类偏好训练 |
| 3. RL 优化 | 使用 PPO 优化策略 |

### DPO（直接偏好优化）

RLHF 的简化替代方案：

| 方面 | RLHF | DPO |
|--------|------|-----|
| 奖励模型 | 需要 | 不需要 |
| PPO | 需要 | 不需要 |
| 复杂性 | 高 | 低 |
| 稳定性 | 中等 | 高 |
| 质量 | 高 | 相当 |

### 奖励建模

| 组件 | 描述 |
|-----------|-------------|
| 偏好数据 | 人类对响应的排名 |
| 奖励模型 | 预测人类偏好分数 |
| Bradley-Terry | 偏好的概率模型 |
| 损失函数 | 成对排名损失 |

## RAG（检索增强生成）

### RAG 架构

| 组件 | 用途 | 技术 |
|-----------|---------|------------|
| 文档加载器 | 摄入文档 | PDF、网页、数据库 |
| 文本分割器 | 分块文档 | 递归、语义 |
| 嵌入模型 | 将文本转换为向量 | Sentence Transformers |
| 向量存储 | 存储嵌入 | ChromaDB、Pinecone、FAISS |
| 检索器 | 查找相关块 | 相似度搜索 |
| 生成器 | 生成答案 | LLM |

### RAG 流程

1. **加载** 来自来源的文档
2. **分割** 为可管理的块
3. **嵌入** 块为向量
4. **存储** 向量到向量数据库
5. **查询** 用户问题
6. **检索** 相关块
7. **生成** 带上下文的答案

### 分块策略

| 策略 | 块大小 | 重叠 | 使用场景 |
|----------|-----------|---------|----------|
| 固定大小 | 500-1000 token | 50-100 | 通用 |
| 递归 | 可变 | 10-20% | 结构化文本 |
| 语义 | 按含义 | 不适用 | 复杂文档 |
| 文档 | 按章节 | 不适用 | 结构良好的文档 |

### 向量数据库

| 数据库 | 类型 | 功能 |
|----------|------|----------|
| ChromaDB | 开源 | 简单、本地 |
| Pinecone | 云 | 托管、可扩展 |
| FAISS | 开源 | 高性能 |
| Weaviate | 开源 | 混合搜索 |
| Qdrant | 开源 | 基于 Rust |

## 总结

大型语言模型代表了自然语言处理的重大进步。关键要点：

1. **Transformer** 是现代 LLM 的基础
2. **预训练** 在大型语料库上创建通用能力
3. **微调** 将模型适应到特定任务
4. **RLHF/DPO** 将模型与人类偏好对齐
5. **提示工程** 最大化模型能力
6. **RAG** 将模型锚定在外部知识中
7. **智能体** 实现复杂的多步骤任务
8. **量化** 实现高效部署
9. **评估** 衡量模型能力
10. **部署** 需要仔细优化

该领域持续快速发展，新的架构、训练技术和应用不断涌现。
